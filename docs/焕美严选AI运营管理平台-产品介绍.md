## 一、产品概述

### 1.1 什么是焕美严选AI运营管理平台？

焕美严选AI运营管理平台是一套专为电商企业打造的**一站式智能化运营解决方案**。我们将 AI 技术深度融入企业运营的各个环节——从客户接待、产品销售到数据分析——让 AI 成为企业最可靠的"数字员工"。

### 1.2 我们的差异化定位

| 传统方案 | 我们的方案 |
|---------|-----------|
| 客服系统与官网分离 | AI 官网 + 智能客服一体化 |
| 数据分析需要专人操作 | AI 自动解读，即问即答 |

## 四、模块三：AI 员工矩阵

### 4.1 什么是 AI 员工？

AI 员工是具有**独立身份、专业技能和知识储备**的智能体。它们不是简单的问答机器人，而是能够：

- 理解企业业务逻辑
- 调用内部知识库作答
- 按照既定流程执行任务
- 持续学习优化服务

### 4.2 核心理念

| 理念 | 说明 |
|------|------|
| 🎭 **人格化设计** | 将被动工作的 AI 工具设计成**能主动工作并人格化**的 AI 员工，拥有独立身份和专业形象 |
| 📋 **SOP 技能封装** | 将企业业务逻辑和工作流封装成**技能文件**，AI 员工按 SOP 自动执行 |
| 📈 **持续学习进化** | 长期训练让 AI 员工**越来越熟悉**公司业务，能力持续迭代提升 |

### 4.6 知识库管理（零技术门槛）

运营人员只需在对应目录下**创建或编辑 Markdown 文件**，AI 员工即可自动掌握新知识：

```
agents/
└── knowledge/              ← 知识库根目录
    ├── enterprise/         ← 企业知识
    │   ├── brand.md        → 品牌故事（AI 自动学习）
    │   ├── policy.md       → 退换货政策
    │   └── contact.md      → 联系方式
    ├── products/           ← 产品知识
    │   ├── skincare.md     → 护肤知识库
    │   ├── faq.md          → 常见问题
    │   └── electronics.md  → 数码知识库
    └── operations/         ← 运营知识
        ├── refund_policy.md → 退款流程
        └── shipping.md      → 发货说明
```

**添加新知识的操作示例**：

```markdown
# 退换货政策（编辑 agents/knowledge/enterprise/policy.md）

## 普通商品退换货
- 7天内无理由退换（自签收日起）
- 运费由买家承担

## 特殊商品
- 贴身衣物：不支持退换
- 定制商品：不支持退换
```

编辑保存后，AI 员工小雪即刻掌握此项政策，可回答相关咨询。

### 4.7 AI 员工底层框架

每位 AI 员工的智能表现，源于其底层框架的精心设计。框架由三大核心支柱构成：**身份（Identity）**、**记忆（Memory）**和**技能（Skills）**。

| 支柱 | 说明 |
|------|------|
| **身份（Identity）** | 定义 AI 员工的元数据：姓名、角色、头像、问候语。配置在 `agents/profiles/agents.json` 中。 |
| **记忆（Memory）** | AI 的"知识大脑"。`knowledge_files` 字段指向知识库 MD 文件路径，对话时据此读取对应文件内容作为上下文注入 LLM。 |
| **技能（Skills）** | AI 的"能力插件"。每个技能对应 `agents/skills/skill_*.md` 文件，定义触发词、排除词、工具调用和常见问答规范。 |

#### 身份配置（agents.json）

```json
agents/profiles/agents.json
{
  "id": "xiaoxue",
  "name": "小雪",
  "role": "智能客服专员",
  "avatar": "/images/agents/ai_xiaoxue.png",
  "greeting": "您好！我是小雪，您的专属客服。有什么可以帮您？",
  "knowledge_files": ["products/skincare.md", "products/faq.md"],
  "skills": ["skill_order_query", "skill_product_query", "skill_complaint"]
}
```

其中 `knowledge_files` 字段指向知识库文件路径，AI 对话时据此加载对应 MD 内容作为上下文。

#### 技能配置（skill_*.md）

```markdown
agents/skills/skill_order_query.md
# 技能文件：订单查询

## 触发词
"查订单"、"我的订单"、"发货了吗"

## 排除词
"取消订单" → 触发退款技能，"修改地址" → 转人工

## 工具定义
query_order（根据手机号或订单号查询订单）
get_logistics（获取物流轨迹）

## 常见问答
订单什么时候发货？→ 付款后24小时内发货
```

#### 记忆检索流程（实际代码逻辑）

```
接收用户问题
    │
    ▼
意图识别（IntentRecognizer）
→ 关键词匹配：订单/发货/物流 → order_query（置信度 0.0-1.0）
    │
    ▼
加载知识文件
→ 根据 agents.json 中 knowledge_files 字段，读取对应 MD 文件内容
    │
    ▼
组装系统提示词
→ 系统提示词模板 + 知识库上下文 + 对话历史（最近10轮）→ LLM
    │
    ▼
流式输出 → Qwen3.5 Plus 返回，流式输出到前端
```

> **注**：知识库检索当前为**关键词匹配**模式（非向量检索），根据 agent 配置的 knowledge_files 列表直接读取对应 MD 文件。
